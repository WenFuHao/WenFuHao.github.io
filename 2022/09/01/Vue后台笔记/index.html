Vue后台笔记

发表于2022-09-01|更新于2022-09-06|VueVue
|字数总计:766|阅读时长:2分钟|阅读量:|评论数:

1.对于系统事件以外不能直接用click等进行事件处理，因为他是自定义事件。

解决方案：Vue2可用修饰符@click.native = “” 把自定义事件变为原生DOM事件

2. 属性修饰符sync【组件通信方式一种】：

可以实现父子组件数据同步

3. $listeners与$attrs(组件通信方式一种)

他们两者是组件实例的属性，可以获取到父组件给子组件传递props与自定义事件

4. $children与$parent:

ref可以获取到某一个组件，子组件
$children组件的属性，可以获取到当前组件的全部子组件【数组】
$parent组件实例的属性，可以获取到当前子组件的父组件，进而可以操作父组件的数据与方法

5. 混入mixin

如果项目中出现很多结构类似功能，想到组件复用
如果项目当中很多的组件js业务逻辑相似，想到mixin【可以把多个组件js部分重复，相似的地方】

6. 插槽：可以实现父子组件通信（通信的结构）

项目笔记：

1. 解决跨域问题：

在vue配置文件中把原先的mock数据修改before：require(‘./mock/mock-serve.js’)为：

// 配置proxy代理跨域
proxy: {
‘/dev-api’: {
target: ‘http://localhost:3000',
pathRewrite: { ‘^/dev-api’: ‘’ }
}
}

2. ElementUI的使用

表格组件：
data: 表格组件将来需要展示的数据—–数组类型
boder: 是给组件添加边框
column属性
label：显示标题
width：对应标题的宽度
align：标题的对齐方式
prop: 对应列内容的字段名

4. 接口跨域问题2：

1.将原来的/dev-api改为/dev-api/admin/acl
2.复制一份该跨域配置，改为/dev-api/admin/product
3.将新的配置的端口号从8170改为8510
4.其实就是针对登录和获取trademar分别做了代理

5. 注意：ElementUI的table组件，展示的数据是以一列一列进行展示数据

6. 对话框：

  1. title: 左上角对应信息
  2. :visible.sync:控制对话框显示与隐藏用的
  3. :visible.sync=”dialogFormVisible” 对话框显示隐藏的属性

7. 完成添加品牌的功能

  1. 书写相关APi接口
  2. 前台需要收集数据。给服务器提交数据(发送相关请求)
  3. 图片action代表图片上传地址
  4. 参数：
    :on-success = handleAvatarSuccess(res,file)可以检测到图片上传成功，当图片上传成功会执行一次
    res: 上传成功之后返回前端数据
    file: 上传成功之后服务器返回的前端数据
    ::before-upload 可以在图片上传之前要做的什么事
